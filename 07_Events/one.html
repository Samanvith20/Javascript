//Events // events are actions or occurrences that happen in the browser, such as a user clicking a button, resizing the window, or submitting a form //Event Listener: An event listener is a piece of code that "listens" for a specific event to occur. //Callback Function: The code that gets executed when an event occurs is often called a "callback function Event Example Event Types: Click Event: Triggered when a mouse button is clicked. Key Events: Triggered when a key is pressed or released. Mouse Events: Include events like mouseover, mouseout, mousemove, etc. Form Events: Events related to HTML form elements, like submit and change. Event Handlers: A function or a piece of code that specifies how the program should respond when a particular event occurs. addEventListener: A method used to attach an event listener to an element. It takes the event type and a callback function as parameters Example: element.addEventListener('click', function() { // Your code here }); Event object: The event object in JavaScript is automatically generated when an event occurs and is passed to the event handler. element.addEventListener('click', function(event) { // Access event properties, e.g., event.target }); Removing Event Listeners: It's important to remove event listeners when they are no longer needed to avoid memory leaks. example: var myButton = document.getElementById('myButton'); myButton.addEventListener('click', function() { alert('Button clicked!'); myButton.removeEventListener('click', function() { alert('Button clicked!'); }); }); Event Bubbling and Capturing: Eveent Bubbling: from low to high (The event starts from the target element and moves outward towards the outermost ancestor.) Capturing Phase: The event starts from the outermost ancestor and moves towards the target element.[high to low] we can stop the any event with stopPropagation() method. example: htnml Events
